Script usage.
The alert allows to execute script on the server when triggered.

Add a script

Check the script checkBox to enable the option. Then choose your script. The script must have a valid shebang. It will be upload when saving the alert.

Remove the script

If you uncheck the box, the script will be delete when saving the alert.

Arguments

The Plugin can send to the script some data in arguments. (Some argument may not be available)
idAlert
id Elasticsearch of the alert
idHistory
id Elasticsearch of the history created
idCron
id Elasticsearch of the cron
date
date of trigger
alert
json of the alert object base64 encoded
message
list of message send by the alert in JSON base64 encoded
result
json of the result of the request perform by alert base64 encoded
status
HTTP code of the request
cron
json of the cron object base64 encoded
./script --idAlert [id] --date [data] --alert base64(JSON([alert])) ...
More information in the doc.
